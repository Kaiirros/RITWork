What is the DOM

Simple!

  • DOM = Document Object Model
  • Document = the web page
  • Object = things on the page
  • Model = standard way of naming and accessing the things

 

How is it created?

The Document Object Model (DOM) is an object model for HTML, XML, SVG (and other) documents.

  • When a web page loads, the browser creates an object model of that page.
  • That model provides objects with properties and methods of all HTML elements.
    • Inclusive of things like childNodes and parentNodes, so we can access them via a tree structure.
  • Web scripts can access those objects to change, add, or delete HTML elements and their attributes.

DOM Node Tree

DOM is organized as a tree of nodes. In DOM, everything is a node!

  • The entire document is a document node
  • Every HTML element is an element node
  • The text inside HTML elements are text nodes
  • Every HTML attribute is an attribute node
  • Comments are comment nodes

Some examples

If I wanted to find (in any page):

  • Get the second paragraph and set the CSS to display:none
    • document.getElementsByTagName(“p”)[1].style.display=”none”;
  • Get the third link on the page and set the link to “http://www.rit.edu”
    • document.getElementsByTagName(“a”)[2].href=”http://www.rit.edu”;
  • Find an image with the id=‘best’ and give it a classname of “changed”
    • document.getElementById(‘best’).class=”changed”;

 

In short, the simple way to think about things:

  • English:
    • (find the element I want).attribute = new value
  • Code
    • document.getElementById(‘best’).class=”changed”;

Can even take this to events:

  • document.getElementById(‘id’).onclick=function(){
            //some code I want to execute on click
    };
